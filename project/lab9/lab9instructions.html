Lab 9 - JavaScript, JSON and AJAX

In lab 3 we made a website which consisted of a homepage (index.html) and a projects page. Since then we have been adding projects to that page for each lab.

Well, here we go again. We are going to modify the projects page we created in that lab again, but this time in a different way.

In this lab,
  • You are to make a JSON or XML file (your choice) with a list of your project menu items

    I have provided a JSON template to get you started.

    I have also provided sample of Lab 7 from a previous year as an example of how to use AJAX to read a file

  • You are to write some JavaScript to read that file and create your projects menu dynamically for your website.

    Your javascript file must be external to your HTML file.

In order to test this file with Chrome when the website is loaded on your PC/Mac, you will need to run this from your localhost by starting your webserver and making sure the content is located on your local webserver (www/iit, htdocs/iit, etc)

The reason for this has to do with the Same Origin Policy protection in Chrome

Also note that if you are using Brackets or VSCode Live Server, you may be able to test this in the integrated browser from them without running it from your local webserver.

This has to do with Same Origin Policy as discussed in class - More on this in the lab class

While it may work with the command arguments discussed previously, you will not need these flags when running your site from your local AMP or RCS deployment.

When you are done, place a link to the Homepage of your website into your README file in the root of your lab9 folder prior to zipping it up and submitting on LMS.

You will be graded according to the following criteria: Lab 9 Rubric

NOTE: Going forward through the end of this semester and forward you will be able to simply add records into the JSON (or XML) file and have your menu be updated accordingly without have to again edit your projects page.
